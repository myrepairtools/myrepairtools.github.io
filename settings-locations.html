Adding a Location

The store list lives in assets/locations.js — one source of truth every page reads. Adding a store is a small code change plus systems setup, best done together in a Claude session:

  1. Get the store's exact RepairQ name (canonical names must match RepairQ exports).
  2. Ask Claude to add it to locations.js (name, display label, order, aliases).
  3. Work through the Opening a New Store — systems setup checklist in the Knowledge Base (RingCentral line, QBO accounts & class mapping, Square location, QB Time, permissions).

Editing locations in place here (a Supabase-backed list) is on the roadmap — today this page is the read-only registry.
