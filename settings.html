RingCentral SMS
Connection
Checking…

Customer texting (Ready-for-Pickup, follow-ups) goes out through our own RingCentral pipe — one app, one Personal JWT per store user. Credentials live server-side as Supabase secrets; this page only shows status. To connect a store: sign in at developers.ringcentral.com as that store’s RingCentral user, mint a Personal JWT authorized for our app, and add it as the Supabase secret named below.

Store lines

Each store texts from its own number. A store whose JWT isn’t connected yet falls back to the default line so texts never fail — it just sends from the wrong store until connected. Store names are the official RepairQ names; other systems’ spellings resolve through aliases.

Message templates

The wording customers get for automated texts. Short codes fill in per send. Set a shared default that every store inherits, or give a store its own wording (e.g. different hours or tone).

for
Short codes: {name} {device} {store} {tech} {hours} — {name} = customer’s first name · {tech} = signed-in RepairQ user · {hours} = today’s store hours · {location} also works for {store}. Unused codes are fine.
Response-time alerts

A backend watchdog checks every few minutes: if a customer’s text goes unanswered for the time below, the people listed here get a text. Replies made anywhere (our panel or the RingCentral app) count. Quiet 8pm–8am; short “thanks!”-type closers are skipped; at most one nudge per conversation every 30 min.

Automated calls

Ready-for-Pickup calls (for customers whose follow-up preference is Call) go out through Twilio but present the store’s own RingCentral number as caller ID — once that number is verified. Until then those calls fall back to a generic number. Verify each line below: Twilio will ring the store phone and read a 6‑digit code — have someone standing at that store’s phone, then type the code it reads into the phone keypad. If an auto-attendant menu answers the line, give the verify prompt the keys a caller presses to reach a person (e.g. ww1 — w = half-second pause) so the call dials through the menu instead of getting stuck in it. If the menu can’t be beaten by keypress, temporarily set that store’s RingCentral auto-receptionist to ring a desk phone directly, verify, then put it back — it’s one-time per store.

QuickBooks Online
Connection
Checking…

The Cash Journal posts each store’s month-end cash revenue straight into QuickBooks Online as a journal entry (debit Cash on Hand, credit Store Revenue). Connecting is owner-only and the credentials stay server-side — the browser never sees them.

Secrets not set — one-time setup:
  1. Create an app at developer.intuit.com with the Accounting scope.
  2. Set the QBO_CLIENT_ID and QBO_CLIENT_SECRET function secrets in Supabase.
  3. Register the redirect URI https://xuvsehrevxackuhmbmry.supabase.co/functions/v1/qbo in the app.
  4. Come back here and hit Connect.
Account Mapping

Each store’s journal entry debits its Cash on Hand account and credits its Store Revenue account. Pick the matching QBO accounts per store — the Cash Journal refuses to post a store until it’s mapped.

Each store has its own cpr.parts account, so each store connects separately. Connect opens the cpr.parts sign-in — log in with that store’s account and the access token is stored server-side (the browser never sees it; tokens don’t expire unless revoked). Sign out of cpr.parts, or use a private window, between stores so each sign-in uses the right account.
